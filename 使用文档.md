# Hugo 博客使用文档

## 站点信息

- 站点目录：`D:\ClaudeOutput\my-site\`
- 文章目录：`content\posts\`
- 配置文件：`hugo.toml`
- 主题：mytheme（可替换）
- 本地预览：http://localhost:1313/

## BAT 工具

| 文件 | 功能 |
|---|---|
| `serve_启动预览.bat` | 启动本地预览服务器，含草稿文章，实时刷新 |
| `build_构建发布.bat` | 构建静态文件到 public/ 目录 |
| `new-post_新建文章.bat` | 输入标题后自动创建 md 文件 |
| `clean_清除输出.bat` | 删除 public/ 目录 |

## 日常写文章流程

```
1. 双击 new-post_新建文章.bat → 输入标题，生成 md 文件
2. 编辑 content/posts/xxx.md → 写文章内容
3. 双击 serve_启动预览.bat → 浏览器打开 http://localhost:1313/ 查看效果
4. 关掉预览（关闭窗口即可）
5. 双击 build_构建发布.bat → 生成 public/ 用于部署
```

## 文章格式

```markdown
+++
date = '2026-04-19T21:35:23+08:00'
draft = false
title = '文章标题'
tags = ['标签1', '标签2']
categories = ['分类']
+++

正文内容，支持 Markdown 语法。
```

- `draft = true`：草稿，预览时需要 `serve.bat` 才能看到
- `draft = false`：正式发布

## 部署到 GitHub Pages

### 首次部署

1. 在 GitHub 创建仓库：`username.github.io`
2. 双击 build_构建发布.bat
3. 打开终端执行：

```bash
cd D:\ClaudeOutput\my-site\public
git init
git remote add origin https://github.com/username/username.github.io.git
git add .
git commit -m "first post"
git push -u origin main
```

4. 在 GitHub 仓库 Settings → Pages → Source 选 main 分支

### 后续更新

```bash
# 写完文章后
双击 build_构建发布.bat
cd public
git add .
git commit -m "new post"
git push
```

## 更换主题

网络通畅时可以下载社区主题，例如 PaperMod：

```bash
cd D:\ClaudeOutput\my-site
git submodule add https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod
```

然后修改 hugo.toml：

```toml
theme = 'PaperMod'
```

## 常见问题

- **端口被占用**：hugo server 默认用 1313 端口，可加 `--port 8080` 换端口
- **文章不显示**：检查 `draft` 是否为 `false`
- **样式没更新**：清除浏览器缓存，或 Ctrl+F5 强制刷新
- **GitHub 推送失败**：检查网络，国内可能需要代理或镜像
